# Практическая работа №8

## Тема работы
Разработка Android-приложения с боковым меню (`Navigation Drawer`) и навигацией между экранами.

## Цель работы
Освоить создание бокового меню в Android Studio, связать пункты меню с экранами приложения и оформить единый шаблон отображения содержимого для каждого раздела.

## Задание
Необходимо было реализовать приложение с выдвижным боковым меню. Каждый пункт меню должен открывать отдельный экран, при этом все экраны должны использовать общий макет, но отображать разное содержимое.

## Проделанная работа
1. Проанализирована структура проекта, созданного на основе шаблона Android Studio.
2. Выявлено несоответствие между пунктами бокового меню и `navigation graph`: в меню были одни идентификаторы, а в навигации использовались шаблонные экраны `Home`, `Gallery`, `Slideshow`.
3. Настроена корректная навигация для всех пунктов меню:
   `Новая группа`, `Новый секретный чат`, `Создать канал`, `Контакты`, `Избранное`, `Звонки`, `Пригласить друзей`, `Настройки`.
4. Удалены лишние шаблонные фрагменты и `ViewModel`, которые не относились к заданию.
5. Создан общий фрагмент `MenuSectionFragment`, который используется для всех экранов меню.
6. Для общего фрагмента разработан единый макет:
   большой значок по центру экрана шириной примерно `2/3` от экрана и подпись под ним.
7. Для каждого пункта меню в `mobile_navigation.xml` переданы собственные параметры:
   заголовок и иконка, соответствующие выбранному пункту меню.
8. Текстовые подписи меню вынесены в `strings.xml`, чтобы ресурсы были оформлены корректно и единообразно.
9. Убраны элементы шаблона, не связанные с заданием:
   `FloatingActionButton` и лишний пункт верхнего меню.

## Результат
В приложении реализовано рабочее боковое меню. При выборе любого пункта открывается экран с одинаковой структурой интерфейса, но с разным содержимым:
- отображается крупная иконка, соответствующая пункту меню;
- под иконкой выводится подпись с названием выбранного раздела.

## Вывод
В ходе работы были закреплены навыки:
- создания `Navigation Drawer`;
- настройки `NavController` и `navigation graph`;
- использования одного фрагмента для нескольких экранов;
- передачи данных в экран через аргументы навигации;
- организации строковых и графических ресурсов Android-приложения.

В результате получено приложение с корректно работающим боковым меню и единым стилем отображения всех разделов.
